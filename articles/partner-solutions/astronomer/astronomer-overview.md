# What is productname – An Azure Native ISV Service?

Azure Native ISV Services enable you to easily provision, manage, and tightly integrate independent software vendor (ISV) software and services on Azure. This Azure Native ISV Service is developed and managed by Microsoft and F5.

You can find productname – An Azure Native ISV Service in the [Azure portal](https://portal.azure.com/#view/HubsExtension/BrowseResource/resourceType/astronomer.astronomerPLUS%2FastronomerDeployments) or get it on [Azure Marketplace](https://azuremarketplace.microsoft.com/marketplace/apps/f5-networks.f5-astronomer-for-azure?tab=Overview).

In this article you learn how to enable deeper integration of the astronomerSaaS service with Azure.

productname delivers secure and high performance applications using familiar and trusted load balancing solutions. Use productname as a reverse proxy within your Azure environment.

The productname offering in the Azure Marketplace allows you to manage astronomer in the Azure portal as an integrated service. You can implement productname as a monitoring solution for your cloud workloads through a streamlined workflow.

You can set up the productname resources through a resource provider named astronomer.astronomerPlus. You can create and manage productname resources through the Azure portal. astronomer owns and runs the software as a service (SaaS) application including the astronomer accounts created.

Here are the key capabilities provided by the productname integration:

- **Seamless onboarding** of productname software as an integrated service on Azure.
- **Unified billing** of productname through Azure monthly billing.
- **Single-Sign on to productname** - No separate sign-up needed from astronomer portal.
- **Lift and Shift config files** - Ability to use existing Configuration (.conf) files for productname deployment.

## Pre-requisites for productname

### Subscription owner

The productname integration can only be set up by users who have Owner access on the Azure subscription. Ensure you have the appropriate Owner access before starting to set up this integration.
